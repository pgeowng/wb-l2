Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
В первом тесте в defer изменяется значение возвращаемой переменной.
Во втором defer меняет значение локальной переменной.
Когда исполняется return x, значение переменной x копируется в ячейку возвращаемой переменной int.
Поэтому изменения локальной x никак не влияют на результат.

Конструкция defer откладывает выполнение функции или метода, до return, или выхода из функции, или до panic.
Как только исполняется одно из условий, вызываются все defer в обратном порядке к тому, в котором были вызваны.

func fn() {
	defer func(){fmt.Println(1)}()
	defer func(){fmt.Println(2)}()
}
fn()
// 2
// 1

Параметры вызова исполняются сразу и ожидают до вызова.

```
